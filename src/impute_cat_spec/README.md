# Imputation Pipeline

This package implements a multi-stage machine learning pipeline designed to extract conversational embeddings, train baseline classifiers, fuse meta-features, and impute missing features (category, specificity). The architecture relies on sequential execution to build up from raw text embeddings to a highly optimized ensemble meta-learner.

---

## Installation & Setup

Before running the pipeline, ensure your environment is configured with the necessary scientific and machine learning libraries.

* **Base Dependencies:** The pipeline requires `polars`, `pandas`, `numpy`, `tqdm`, and `matplotlib` or `seaborn` for data processing and visualization.
* **Machine Learning Libraries:** You must install `scikit-learn`, `lightgbm`, `xgboost`, `optuna`, and `scipy` to support the modeling, tuning, and optimization stages.
* **Embedding Libraries:** The extraction phase requires `sentence-transformers` for deep learning models and `scikit-learn` for TF-IDF vectorization.

---

## Data Requirements & Directory Structure

The pipeline only requires the raw input datasets to be present prior to execution. All other directories (for embeddings, baseline predictions, and model outputs) are automatically generated by the scripts as they run.

* **Input Data:** The raw datasets must be strictly located in the `../../../data/talkpl-ai/` directory across the respective challenge subdirectories.
* **Model Caching:** HuggingFace models can be pre-downloaded and cached to allow for air-gapped execution on compute nodes.

### Expected Directory Tree

Below is the required structure for the input data. The pipeline will automatically build out the `impute_values` folder and its subdirectories during execution:

## Pipeline Architecture & Execution Commands

**CRITICAL:** The scripts comprising this pipeline must be executed one after the other in the exact order listed below. Each step depends on the data generated by the previous one.

### Step 0: Model Initialization

The system fetches and caches required HuggingFace sentence transformer models locally.

```bash
python 0_download_models.py --model all

```

### Step 1: Embedding Extraction

Conversational turns are flattened and processed into dense vector embeddings. You must run this script for each model you wish to extract.

```bash
python 1_extract_embeddings.py --model tfidf
python 1_extract_embeddings.py --model sbert-minilm
python 1_extract_embeddings.py --model bge-large
python 1_extract_embeddings.py --model mpnet
python 1_extract_embeddings.py --model e5-large
python 1_extract_embeddings.py --model mxbai
python 1_extract_embeddings.py --model qwen3-embed-0.6b
python 1_extract_embeddings.py --model qwen3-embed-4b

```

### Step 2: Baseline Predictions

The system aggregates embeddings and sweeps them across classifiers to generate baseline predictions.

```bash
python 2_baseline_predictions.py --model all --use_gpu

```

### Step 3: Diversity Analysis

Baseline predictions are evaluated, mapped for diversity using Cohen's Kappa, and passed through a greedy oracle selection to generate a recommended ensemble list.

```bash
python 3_filter.py

```

### Step 4: Hyperparameter Tuning

The pipeline utilizes Optuna and 5-Fold Stratified Cross-Validation to find optimal parameters. You will run this command for the specific models recommended by Step 3.

```bash
python 4_hyperparameter_tuning.py --target category --emb_model bge-large --agg_method mean --clf_model lgbm

```

### Step 5: Meta-Feature Generation

The best-tuned models are trained on the data to generate meta-feature matrices containing hard predictions and soft probabilities.

```bash
python 5_meta_feature_generation.py

```

### Step 6: Ensemble Fusion

The system applies mathematical heuristics and meta-learner stacking to the meta-features to create final predictions.

```bash
python 6_fusion_models.py

```

### Step 7: Evaluation & Extraction

The ensemble's performance is validated against known labels in Blind A, and the top-performing model's predictions are extracted into JSON files.

```bash
python 7_evaluate_blind_a.py

```

### Step 8: Imputation Validation

The system checks data quality between common users in Blind A and B, merges the JSON imputations, and generates the finalized dataset.

```bash
python 8_imputate_blind_b.py

```

## Configuration Parameters

The generation and tuning constraints are hardcoded at the top of the respective scripts.

* **Target Variables:** The pipeline independently processes and predicts the `category` and `specificity` targets.


* **Competence Gating:** Models must achieve at least 85% of the best model's F1 score to be included in the ensemble.


* **Ensemble Size:** The greedy oracle selection caps the maximum ensemble size at 25 models.


* **Cross-Validation:** Optuna hyperparameter tuning runs using 100 trials and 5 cross-validation folds with a fixed random state of 42.



---

## Output and Visualization

The pipeline automatically generates evaluation metrics, visualizations, and finalized datasets for submission.

* **Hierarchical Clustermaps:** The diversity analysis step generates a Cohen's Kappa clustermap showing model similarity, saved to the `plots` directory.


* **Leaderboard Visualizations:** Bar charts ranking the macro-F1 scores of individual models and final ensemble strategies are exported as PNG files.


* **Prediction JSONs:** The extracted predictions for the top-performing ensemble are saved as `final_blind_a_predictions.json` and `final_blind_b_predictions.json`.


* **Final Parquet:** The pipeline concludes by writing out `final_blind_b.parquet`, combining recovered Blind A data with the new Blind B imputations.
